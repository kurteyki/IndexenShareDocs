Overview Last updated: Monday 07 Maret 2022

This Documentation describes the installation and configuration steps of IndexenShare.

While the installation process is rather simple, some of the instructions presented here will require minimum understanding in working with databases and managing files on a server.

If you're not familiar with the procedures presented, we do offer installation Services for an additional fee.

Requirements

Please ensure that your server meets the following minimum requirements:

Software Modules
PHP 7 Fileinfo, JSON, Mbstring, OpenSSL, PDO, Tokenizer, XML, GD, cURL, and ALl required of Vendor
Apache
MySQL
API Modules
Google API Key Indexing API, Google Search Console API

Installation

Uploading the files

On some operating systems, the dotfiles are hidden by default. Before starting to upload the files, please make sure your file explorer has the option to view hidden files turned on.

Upload what's inside the Script folder on the web root folder of your domain. This folder is generally called public_html, html, or example.com, but depending on your server's configuration, it might be named differently.

Useful resources
cPanel - File Manager

Creating and configuring the database

  1. Create a new MySQL database.
  2. Create a new MySQL username and password.
  3. Assign full privileges to your database username.
  4. Upload database from files Database/indexenshare.sql
  5. Config Database on includes/database.php
Useful resources
cPanel - MySQL Database Wizard

Create and configuring Google API Key

  1. Create API on Google Cloud Platform
  2. Enable Indexing API
  3. Enable Google Search Console API

  4. Configuration OAuth consent screen

    step : OAuth consent screen

    1. Insert App name
    2. Insert User support email
    3. Insert Developer contact information
    4. Click button Save and Continue

    step : Scopes

    1. Click Button Add or Remove Scopes
    2. Check userinfo.email
    3. Check userinfo.profile
    4. Click button UPDATE
    5. Click button Save and Continue

    last step

    after configuration all step button Save and Continue and back to dashboard

  5. set Publishing status to Production on OAuth consent screen

  6. Create Cerdentials

    1. Insert Application type Web Application
    2. Insert Name

    3. insert Authorised redirect URIs

      if you place this on root domain insert like this https://yourwebsite.com/

      if you place this on sub domain insert like this https://subdomain.yourwebsite.com/

      if you place this on path domain insert like this https://yourwebsite.com/indexenshare/

    4. Click button Create
  7. Copy client_id and client_secret
  8. Config API Key on includes/api.php
  9. Dont Forget to set $quota_indexing_publish to your limit api (default api is 200 request / day)
